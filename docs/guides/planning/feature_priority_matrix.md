**작성일**: 2025-08-09

## 📊 우선순위 매트릭스 개요

### 평가 기준

**비즈니스 임팩트 (Business Impact)** - 1-5점 척도
- **5점 (매우 높음)**: 직접적 매출 증대, 운영비용 대폭 절약, 경쟁 우위 확보
- **4점 (높음)**: 사용자 만족도 크게 향상, 업무 효율성 대폭 개선
- **3점 (보통)**: 기본적인 업무 개선, 사용자 편의성 향상
- **2점 (낮음)**: 부가적 기능, 일부 사용자에게만 유용
- **1점 (매우 낮음)**: 선택적 기능, 장기적으로만 효과

**기술 복잡도 (Technical Complexity)** - 1-5점 척도
- **5점 (매우 높음)**: 새로운 기술 스택, 복잡한 알고리즘, 높은 성능 요구사항
- **4점 (높음)**: 기존 시스템 대규모 수정, 복잡한 비즈니스 로직
- **3점 (보통)**: 일반적인 CRUD + 추가 로직, 적당한 데이터 모델링
- **2점 (낮음)**: 단순한 CRUD 작업, 기존 패턴 활용
- **1점 (매우 낮음)**: 간단한 UI 변경, 설정 추가

## 💡 의사결정 가이드라인

### 우선순위 결정 원칙
1. **MVP 완성 최우선**: ClassFlow 기반 안정화 후 확장
2. **ROI 기반 선택**: 투자 대비 효과가 명확한 기능 우선
3. **단계적 구현**: 복잡한 기능은 단계별로 나누어 구현
4. **시장 검증**: 사용자 피드백을 통한 지속적 우선순위 조정

### 리스크 완화 전략
- **기술적 리스크**: 복잡한 기능은 PoC(Proof of Concept) 선행
- **비즈니스 리스크**: 시장 수요 검증 후 본격 개발
- **자원 리스크**: 핵심 기능 우선, 부가 기능 후순위

### 성공 지표
```yaml

  - 학원 운영 효율성 300% 향상

Phase 6 완료 시:
  - 시장 점유율 20% 이상
  - 추가 매출원 확보

Phase 7+ 완료 시:
  - AI 기반 교육 플랫폼 완성
  - 글로벌 진출 준비 완료
```
